☕ update-javaエイリアスの詳細

エイリアスコマンド

update-java

☕ Javaとは

Javaは、Webアプリ・Androidアプリ・業務システムなど幅広い用途で使われるプログラミング言語および実行環境です。Javaで作られたプログラムを動かすには「JDK（Java Development Kit）」のインストールが必要です。

🔒 なぜアップデートが必要か

  • セキュリティ: 古いバージョンには脆弱性が存在することがあり、定期的な更新で安全性を保てます
  • 互換性: 新しいライブラリやフレームワークは最新のJavaバージョンを要求することがあります
  • パフォーマンス: 新バージョンでは実行速度や省メモリが改善されています

🔍 コマンドの動作説明

Apple Silicon（M1/M2）とIntel Macを自動判定し、適切なJavaをインストール・更新するコマンドです。

💡 使用方法

  1. ターミナルでupdate-javaコマンドを実行
  2. システムのアーキテクチャが自動判定される
  3. 該当するJavaをインストール・更新

⚙️ 特徴

  • Apple Silicon (ARM64)とIntel (x86_64)を自動判定
  • 各アーキテクチャに最適なJavaをインストール
  • 既にインストール済みの場合は更新を実行

📋 元のコード

update-java() {
  if [ -d "/opt/homebrew/opt/openjdk" ]; then
    echo "Apple Silicon Macとして実行"
    brew install openjdk && sudo ln -sfn /opt/homebrew/opt/openjdk/libexec/openjdk.jdk /Library/Java/JavaVirtualMachines/openjdk.jdk && echo "export PATH=\"/opt/homebrew/opt/openjdk/bin:\$PATH\"" >> ~/.zshrc && source ~/.zshrc && java -version
  else
    echo "Intel Macとして実行"
    brew install openjdk && sudo ln -sfn /usr/local/opt/openjdk/libexec/openjdk.jdk /Library/Java/JavaVirtualMachines/openjdk.jdk && echo "export PATH=\"/usr/local/opt/openjdk/bin:\$PATH\"" >> ~/.zshrc && source ~/.zshrc && java -version
  fi
}

🖼️ herowebpエイリアスの詳細

エイリアスコマンド

herowebp

🔍 コマンドの動作説明

jpg/jpeg/png/heic形式の画像を3つのサイズのWebP形式に自動変換します。ヒーロー画像などレスポンシブで複数サイズが必要な場合に便利です。

💡 使用方法

  1. カレントディレクトリの画像を配置
  2. ターミナルでherowebpコマンドを実行
  3. 以下の3サイズのWebP画像が自動生成されます：
    • 1920x1080px
    • 1200x675px
    • 800x450px

⚙️ 対応形式

  • 入力形式: jpg, jpeg, png, heic
  • 出力形式: WebP (複数サイズ)

📋 元のコード

herowebp() {
  local quality=50
  local method=6
  if [ ! -d "$HOME/editing" ]; then
    echo "Error: ~/editing directory not found"
    return 1
  fi
  cd "$HOME/editing" || return 1
  for img in *.{jpg,jpeg,png,heic}; do
    if [ -f "$img" ]; then
      base="${img%.*}"
      echo "Processing: $img"
      magick "$img" -resize 1920x1080 -quality $quality -define webp:method=$method "${base}_1920x1080.webp"
      magick "$img" -resize 1200x675 -quality $quality -define webp:method=$method "${base}_1200x675.webp"
      magick "$img" -resize 800x450 -quality $quality -define webp:method=$method "${base}_800x450.webp"
      echo "✓ Generated WebP files for: $img"
    fi
  done
}

🖼️ webp50エイリアスの詳細

エイリアスコマンド

webp50

🔍 コマンドの動作説明

jpg/jpeg/png/heic形式を自動検出して、高圧縮率のWebPに変換します。品質50とmethod=6（最高品質の圧縮）で最適なバランスを実現します。

💡 使用方法

  1. カレントディレクトリの画像を配置
  2. ターミナルでwebp50コマンドを実行
  3. 同じフォルダにWebP形式の画像が生成されます

⚙️ 圧縮オプション

  • quality=50: 品質レベル (0-100、50は標準的な圧縮)
  • method=6: 圧縮方法 (0-6、6は最高品質)

📋 元のコード

webp50() {
  local quality=50
  local method=6
  local count=0
  if [ ! -d "$HOME/editing" ]; then
    echo "Error: ~/editing directory not found"
    return 1
  fi
  cd "$HOME/editing" || return 1
  for file in *.{jpg,jpeg,png,heic}; do
    if [ -f "$file" ]; then
      local base="${file%.*}"
      if [ -f "${base}.webp" ]; then
        echo "⊘ Already exists: ${base}.webp (skipped)"
        continue
      fi
      echo "Converting: $file → ${base}.webp"
      magick "$file" -quality $quality -define webp:method=$method "${base}.webp"
      if [ $? -eq 0 ]; then
        echo "✓ Converted: $file"
        ((count++))
      else
        echo "✗ Failed to convert: $file"
      fi
    fi
  done
  echo ""
  if [ $count -gt 0 ]; then
    echo "✓ Successfully converted $count file(s) to WebP"
  else
    echo "⊘ No new files to convert"
  fi
}

🖼️ img2jpegエイリアスの詳細

エイリアスコマンド

img2jpeg

🔍 コマンドの動作説明

複数の画像形式をJPEGに統一して変換します。長辺を1080pxにリサイズするため、ウェブ用途に最適なサイズになります。

💡 使用方法

  1. カレントディレクトリの画像を配置
  2. ターミナルでimg2jpegコマンドを実行
  3. 同じフォルダにJPEG形式（長辺1080px）の画像が生成されます

⚙️ 変換設定

  • 出力形式: JPEG
  • 最大長辺: 1080px
  • アスペクト比: 維持

📋 元のコード

img2jpeg() {
  setopt nullglob
  local count=0
  local failed=0
  local total=0

  for file in *.tiff *.tif *.png *.jpeg *.jpg *.heic; do
    [ -f "$file" ] || continue
    local base="${file%.*}"
    ((total++))

    echo "Converting: $file → ${base}.jpg"
    sips -s format jpeg -Z 1080 "$file" --out "${base}.jpg" &>/dev/null

    if [ $? -eq 0 ]; then
      echo "✓ $(basename "$file")"
      ((count++))
    else
      echo "✗ $(basename "$file")"
      ((failed++))
    fi
  done

  echo ""
  if [ $total -eq 0 ]; then
    echo "⊘ No image files found (tiff, tif, png, jpeg, jpg, heic)"
  elif [ $count -gt 0 ]; then
    echo "✓ Successfully converted $count file(s) to JPEG"
  fi
  if [ $failed -gt 0 ]; then
    echo "✗ Failed to convert $failed file(s)"
  fi
}

🖼️ avifエイリアスの詳細

エイリアスコマンド

avif

🔍 コマンドの動作説明

カレントディレクトリ以下を再帰的に検索し、HEIC/HEIF/JPG/JPEG/PNG/TIFF/TIF/BMP/GIF 形式の画像を AVIF 形式に一括変換します。sips で一時 PNG に変換後、avifenc で品質60の AVIF を生成。変換成功後は元ファイルと一時 PNG を削除します。既に同名の .avif が存在する場合はスキップします。

💡 使用方法

  1. 変換したい画像が含まれるフォルダへ移動
  2. ターミナルでavifを実行
  3. サブフォルダを含む全画像が AVIF 形式に変換されます

⚙️ 変換設定

  • 入力形式: HEIC, HEIF, JPG, JPEG, PNG, TIFF, TIF, BMP, GIF
  • 出力形式: AVIF（品質60）
  • 既存の .avif はスキップ
  • 変換成功後に元ファイル・一時 PNG を自動削除
  • 必要ツール: sips（macOS 標準）、avifenc（要インストール）

📋 元のコード

find . -iname "*.heic" -o -iname "*.heif" -o -iname "*.jpg" -o -iname "*.jpeg" -o -iname "*.png" -o -iname "*.tiff" -o -iname "*.tif" -o -iname "*.bmp" -o -iname "*.gif" | while read f; do
  tmp="${f%.*}_tmp.png"
  out="${f%.*}.avif"
  [[ -f "$out" ]] && continue
  sips -s format png "$f" --out "$tmp" && \
  avifenc -q 60 "$tmp" "$out" && \
  rm "$tmp" "$f"
done

✂️ trim95関数の詳細

コマンド

trim95

🔍 コマンドの動作説明

カレントディレクトリの .heic ファイルを中央から95%サイズにクロップします。macOS 標準の sips を使用するため、ImageMagick 不要。元ファイルはそのまま残し、trimmed_ プレフィックスを付けて出力します。

💡 使用方法

  1. クロップしたい HEIC 画像が含まれるフォルダへ移動
  2. ターミナルで trim95 を実行
  3. 各ファイルの trimmed_元ファイル名.heic が生成されます

⚙️ 処理内容

  • 入力形式: HEIC
  • クロップ率: 縦横それぞれ 95%（中央基準）
  • 出力: trimmed_元ファイル名.heic（元ファイルは保持）
  • 必要ツール: sips（macOS 標準搭載、追加インストール不要）

📋 関数の設定（初回のみ）

🔧 ~/.zshrc への追加

以下のコマンドを実行すると関数が登録されます:

cat >> ~/.zshrc << 'EOF'
trim95() {
  for file in *.heic; do
    [ -f "$file" ] || continue
    read w h < <(sips -g pixelWidth -g pixelHeight "$file" | awk '/pixelWidth/{w=$2} /pixelHeight/{h=$2} END{print w, h}')
    crop_w=$(echo "scale=0; $w * 95 / 100" | bc)
    crop_h=$(echo "scale=0; $h * 95 / 100" | bc)
    offset_x=$(( (w - crop_w) / 2 ))
    offset_y=$(( (h - crop_h) / 2 ))
    sips --cropToHeightWidth $crop_h $crop_w --cropOffset $offset_y $offset_x "$file" --out "trimmed_${file}"
  done
}
EOF
source ~/.zshrc

🖨️ h2pエイリアスの詳細

エイリアスコマンド

h2p

🔍 機能説明

HEIC形式の画像をJPG形式に変換してから、SHARPカラープリンタで自動的にカラー印刷します。画像の向き（縦横）を自動判定し、用紙に最適な方向で印刷します。

💡 使用方法

基本的な使用手順：

  1. 印刷したい HEIC ファイルがあるフォルダでターミナルを開く（またはカレントディレクトリに HEIC ファイルを配置）
  2. 必要に応じて hrsz() で画像をトリム・リサイズ（2480x3508 または 3508x2480）
  3. ターミナルで h2p を実行

🔄 処理フロー

  • HEIC → JPG 変換：ImageMagick (magick) を使用して自動回転対応で変換
  • 画像サイズ判定：変換後のJPGサイズから向きを自動判定
    • 横長（幅 > 高さ）→ 横向き印刷
    • 縦長（幅 ≤ 高さ）→ 縦向き印刷
  • 用紙いっぱい印刷：fit-to-page オプションで拡大印刷
  • カラー印刷：ColorModel=RGB を指定
  • 自動クリーンアップ：印刷完了後、一時JPGファイルを削除

📋 エイリアス定義

h2p() {
  setopt nullglob
  for file in ./*.heic; do
    [ -f "$file" ] || continue
    jpg="${file%.heic}.jpg"
    if ! magick "$file" -auto-orient "$jpg"; then
      echo "✗ エラー: $(basename "$file") の変換に失敗"
      rm -f "$jpg"
      continue
    fi
    dimensions=$(magick identify -format "%w %h" "$jpg")
    width=$(echo $dimensions | cut -d' ' -f1)
    height=$(echo $dimensions | cut -d' ' -f2)
    if [ "$width" -gt "$height" ]; then
      orientation=4
    else
      orientation=3
    fi
    if lp -d SHARP_BP_60C26__3504320300_ -o ColorModel=RGB -o fit-to-page -o orientation-requested=$orientation "$jpg" &>/dev/null; then
      sleep 3
      rm -f "$jpg"
      echo "✓ 処理完了: $(basename "$file")"
    else
      echo "✗ エラー: $(basename "$file") の印刷に失敗"
      rm -f "$jpg"
    fi
  done
}

🖼️ hrszエイリアスの詳細

エイリアスコマンド

hrsz

🔍 機能説明

HEIC形式の画像を正規化サイズにリサイズ・トリムします。画像の向き（縦横）を自動判定し、アスペクト比を損なわないように中央からトリムして、3508x2480 または 2480x3508 のサイズに統一します。

💡 使用方法

基本的な使用手順：

  1. HEIC ファイルがあるフォルダで hrsz を実行
  2. 画像の向きに応じて自動的にサイズを決定
  3. output/ フォルダに処理済み画像を出力

🔄 処理フロー

  • サイズ判定：元画像の幅と高さを取得
    • 横長（幅 > 高さ）→ 3508x2480 にリサイズ・トリム
    • 縦長（幅 ≤ 高さ）→ 2480x3508 にリサイズ・トリム
  • 高品質リサイズ：-density 300 で 300 DPI に設定
  • 中央トリム：-gravity Center で中央からトリム（アスペクト比を損なわない）
  • 拡大対応：-resize ^ で小さい画像も拡大可能
  • ファイル保持：元のファイル名で output/ に出力

📋 エイリアス定義

hrsz() {
  output_dir="output"
  mkdir -p "$output_dir"
  for file in *.heic; do
    width=$(magick identify -format "%w" "$file")
    height=$(magick identify -format "%h" "$file")
    if [ "$width" -gt "$height" ]; then
      magick "$file" -density 300 -gravity Center -resize 3508x2480^ -crop 3508x2480+0+0 +repage "$output_dir/$file"
    else
      magick "$file" -density 300 -gravity Center -resize 2480x3508^ -crop 2480x3508+0+0 +repage "$output_dir/$file"
    fi
    echo "✓ 処理完了: $file"
  done
  echo "✅ 全ファイルが $output_dir フォルダに出力されました"
}

hrsz500

🔍 機能説明

  2. フォルダ内および下位フォルダの全ての画像が処理される
  3. 処理されたファイルは output フォルダに 500x500、72 PPI のHEIC形式で出力
  4. 元のJPEG/PNGファイルは ~/Desktop/original_images/ に移動

  • ディレクトリ作成：~/Desktop/original_images と output フォルダを作成
  • JPEG/PNG処理：
    • 500x500 ピクセルにリサイズ・クロップ（中心基準、背景色は白）
    • 72 PPI で HEIC形式に変換（品質75）
    • output フォルダに出力
    • 元のファイルを original_images/ に移動
  • 既存HEIC処理：
    • 500x500 ピクセルにリサイズ・クロップ（背景色は白）
    • 72 PPI で品質75に統一
    • output フォルダに出力
  • 下位フォルダ対応：find コマンドで再帰的に全ファイルを処理

📋 エイリアス定義

hrsz500() {
  mkdir -p ~/Desktop/original_images
  mkdir -p output

  find . -type f \( -iname "*.jpeg" -o -iname "*.jpg" -o -iname "*.png" \) | while read file; do
    filename=$(basename "$file" | sed 's/\.[^.]*$//')
    output_file="output/${filename}.heic"
    magick "$file" -resize 500x500 -gravity center -background white -extent 500x500 -density 72 -quality 75 "$output_file"
    mv "$file" ~/Desktop/original_images/
    echo "Converted & resized: $(basename "$file") → $(basename "$output_file") (original moved)"
  done

  find . -type f -iname "*.heic" | while read file; do
    filename=$(basename "$file")
    temp_file="output/${filename}.tmp"
    magick "$file" -resize 500x500 -gravity center -background white -extent 500x500 -density 72 -quality 75 "$temp_file"
    mv "$temp_file" "output/${filename}"
    echo "Resized: output/$(basename "$file")"
  done
}

🔢 rnエイリアスの詳細

エイリアスコマンド

rn

🔍 コマンドの動作説明

カレントディレクトリにある画像ファイル（jpg/jpeg/png/tiff/tif/gif/webp/bmp）を対話形式で連番リネームします。開始番号・前後に付ける文字列を指定でき、実行前にプレビューで確認できます。

💡 使用方法

  1. リネームしたい画像があるフォルダに移動
  2. ターミナルでrnを実行
  3. 開始番号を入力して確認
  4. 番号の前後に文字列を追加するか選択（任意）
  5. プレビューを確認してリネーム実行

⚙️ 主な機能

  • 対応形式：jpg / jpeg / png / tiff / tif / gif / webp / bmp
  • 開始番号の入力バリデーション（数字以外はエラー）
  • 番号の前（prefix）または後（suffix）に任意文字列を追加可能
  • リネーム前にプレビュー表示・確認プロンプト
  • mv -iで既存ファイルへの上書きを防止

📋 元のコード

rn() {
  # 開始番号を訊く
  while true; do
    echo -n "開始番号を入力してください: "
    read start
    if [[ -z "$start" || ! "$start" =~ ^[0-9]+$ ]]; then
      echo "エラー: 数字を入力してください。"
      continue
    fi
    echo -n "開始番号は「${start}」でよいですか？ (y/n): "
    read confirm_start
    [[ "$confirm_start" == "y" ]] && break
  done

  # 前後に文字列を付けるか訊く
  local prefix=""
  local suffix=""
  echo -n "番号の前後に文字列を追加しますか？ (y/n): "
  read add_str
  if [[ "$add_str" == "y" ]]; then
    echo -n "文字列を入力してください: "
    read str
    echo -n "番号の「前」ですか「後」ですか？ (前/後): "
    read pos
    if [[ "$pos" == "前" ]]; then
      prefix="$str"
    else
      suffix="$str"
    fi
  fi

  # 対象ファイルを画像から自動検出
  local img_files=()
  for f in *(N-.); do
    [[ "$f" =~ \.(jpg|jpeg|png|tiff|tif|gif|webp|bmp)$ ]] && img_files+=("$f")
  done

  if [[ ${#img_files[@]} -eq 0 ]]; then
    echo "対象の画像ファイルが見つかりません。" >&2
    return 1
  fi

  # プレビュー表示
  echo ""
  echo "【プレビュー】"
  local n=$start
  for f in "${img_files[@]}"; do
    local ext="${f##*.}"
    local new_name="${prefix}${n}${suffix}.${ext}"
    echo "$f → $new_name"
    (( n++ ))
  done

  # 確認
  echo ""
  echo -n "このままリネームしますか？ (y/n): "
  read confirm
  if [[ "$confirm" != "y" ]]; then
    echo "キャンセルしました。"
    return 0
  fi

  # 実行
  n=$start
  for f in "${img_files[@]}"; do
    local ext="${f##*.}"
    local new_name="${prefix}${n}${suffix}.${ext}"
    mv -i -- "$f" "$new_name"
    (( n++ ))
  done
  echo "リネーム完了しました。"
}

エイリアスコマンド

🔍 コマンドの動作説明

カレントディレクトリ自体を1つのZIPファイルにまとめます。ZIPファイルは親ディレクトリに作成されます。

  1. ZIP化したいディレクトリに移動（cd コマンドで対象ディレクトリへ）

⚙️ 処理内容

  • 処理対象: カレントディレクトリ全体
  • エンコーディング: CP932（Windows互換）
  • 出力先: 親ディレクトリ（カレントディレクトリ名.zip）
  • 既にZIPが存在する場合はスキップ

📋 元のコード

zp() {
  local dirname=$(basename "$PWD")
  local zipname="${dirname}.zip"
  local zip_path="$(dirname "$PWD")/$zipname"

  # 既にZIPが存在する場合はスキップ
  if [ -f "$zip_path" ]; then
    echo "⊘ スキップ: $dirname (既にZIPが存在します: $zip_path)"
    return 1
  fi

  echo "処理中: $dirname → $zip_path"

  # ZIPフォーマットをバイナリで直接書き出す（UTF-8フラグ強制セットを回避）
  python3 << PYTHON_EOF
import os, sys, struct, zlib, time

try:
    folder_path = "$PWD"
    zip_path = "$zip_path"
    parent_path = os.path.dirname(folder_path)

    entries = []

    with open(zip_path, 'wb') as zf:
        for root, dirs, files in os.walk(folder_path):
            dirs.sort()
            for file in sorted(files):
                file_path = os.path.join(root, file)
                arcname = os.path.relpath(file_path, parent_path)
                arcname_bytes = arcname.encode('cp932')

                with open(file_path, 'rb') as f:
                    data = f.read()

                compressed = zlib.compress(data, 6)[2:-4]  # deflate
                crc = zlib.crc32(data) & 0xFFFFFFFF
                t = time.localtime()
                dostime = (t.tm_hour << 11) | (t.tm_min << 5) | (t.tm_sec // 2)
                dosdate = ((t.tm_year - 1980) << 9) | (t.tm_mon << 5) | t.tm_mday

                offset = zf.tell()

                # Local file header
                lh = struct.pack('<4sHHHHHIIIHH',
                    b'PK\x03\x04',
                    20,          # version needed
                    0,           # flags: 0 (no UTF-8 flag)
                    8,           # compression: deflate
                    dostime, dosdate,
                    crc,
                    len(compressed),
                    len(data),
                    len(arcname_bytes),
                    0            # extra field length
                )
                zf.write(lh)
                zf.write(arcname_bytes)
                zf.write(compressed)

                entries.append((arcname_bytes, crc, len(compressed), len(data), dostime, dosdate, offset))

        # Central directory
        cd_offset = zf.tell()
        for arcname_bytes, crc, comp_size, uncomp_size, dostime, dosdate, offset in entries:
            cd = struct.pack('<4sHHHHHHIIIHHHHHII',
                b'PK\x01\x02',
                20,          # version made by
                20,          # version needed
                0,           # flags: 0
                8,           # compression
                dostime, dosdate,
                crc,
                comp_size,
                uncomp_size,
                len(arcname_bytes),
                0, 0, 0, 0,  # extra, comment, disk, attr
                0,           # external attr
                offset
            )
            zf.write(cd)
            zf.write(arcname_bytes)

        # End of central directory
        cd_size = zf.tell() - cd_offset
        eocd = struct.pack('<4sHHHHIIH',
            b'PK\x05\x06',
            0, 0,
            len(entries), len(entries),
            cd_size, cd_offset,
            0
        )
        zf.write(eocd)

    sys.exit(0)
except Exception as e:
    print(f"✗ エラー: {e}")
    sys.exit(1)
PYTHON_EOF

  if [ $? -eq 0 ]; then
    echo "✓ 完了: $zip_path"
  else
    echo "✗ エラー: $dirname の変換に失敗しました"
  fi
}

コマンド

p2ha4

🔍 コマンドの動作説明

カレントディレクトリの .pdf ファイルをページごとにHEICへ変換し、A4縦アスペクト比（210:297）で中央クロップします。クロップ割合を引数で指定できます（デフォルト: 100%）。中間ファイルは自動的に削除されます。

⚙️ 処理内容

  • 入力形式: PDF（複数ページ対応）
  • 出力形式: HEIC
  • 解像度: 300dpi
  • アスペクト比: A4縦（210:297）
  • クロップ割合: 引数で指定（デフォルト100%、例: p2ha4 95 で95%）
  • 出力ファイル名: 元ファイル名_0000.heic（ページ番号付き）
  • 必要ツール: ImageMagick（magick）、heif-enc、sips、bc

📋 関数の設定（初回のみ）

🔧 ~/.zshrc への追加

以下のコマンドを実行すると関数が登録されます:

cat >> ~/.zshrc << 'EOF'
p2ha4() {
  local pct=${1:-100}
  for pdf in *.pdf; do
    [ -f "$pdf" ] || continue
    base="${pdf%.*}"
    magick -density 300 "$pdf" -quality 100 "__p2h_${base}_%04d.png"
    for png in "__p2h_${base}"_*.png; do
      [ -f "$png" ] || continue
      stem="${png#__p2h_}"; stem="${stem%.png}"
      tmp_heic="__p2h_${stem}.heic"
      out_heic="${stem}.heic"
      heif-enc "$png" -o "$tmp_heic" && rm "$png"
      read w h < <(sips -g pixelWidth -g pixelHeight "$tmp_heic" | awk '/pixelWidth/{w=$2} /pixelHeight/{h=$2} END{print w, h}')
      max_w=$(( w * pct / 100 ))
      max_h=$(( h * pct / 100 ))
      if (( max_w * 297 >= max_h * 210 )); then
        crop_h=$max_h
        crop_w=$(echo "scale=0; $max_h * 210 / 297" | bc)
      else
        crop_w=$max_w
        crop_h=$(echo "scale=0; $max_w * 297 / 210" | bc)
      fi
      offset_x=$(( (w - crop_w) / 2 ))
      offset_y=$(( (h - crop_h) / 2 ))
      sips --cropToHeightWidth $crop_h $crop_w --cropOffset $offset_y $offset_x "$tmp_heic" --out "$out_heic" && rm "$tmp_heic"
    done
  done
}
EOF
source ~/.zshrc

🎬 v2pエイリアスの詳細

エイリアスコマンド

v2p

🔍 コマンドの動作説明

m4vファイルをmp4ファイルに劣化なしで変換します。m4vとmp4は同じMPEG-4コンテナフォーマットのため、再エンコードを行わずコンテナだけを変更することで、品質を損なわずに高速で変換できます。

💡 M4VとMP4について

  • M4V形式: Apple製品用のMPEG-4ビデオコンテナ（DRM保護機能付きの場合もあります）
  • MP4形式: より汎用的なMPEG-4ビデオコンテナ
  • 内部構造: 両者は本質的に同じコンテナフォーマット
  • 変換方法: -c copyオプションでコンテナだけを変更し、ビデオ・オーディオの再エンコードを行わない

💡 使用方法

フォルダを右クリックして「新規ターミナル」を選択すると、そのフォルダに移動した状態でターミナルが起動します。その後、ターミナルでv2pと入力して実行してください。

  1. m4vファイルがあるフォルダで右クリック
  2. 「新規ターミナル」を選択（フォルダに移動した状態で起動）
  3. ターミナルでv2pと入力して実行
  4. 同じフォルダにmp4ファイルが生成されます

⚙️ 処理内容

  • 対象: カレントディレクトリ内のすべてのm4vファイル
  • 処理方式: 再エンコードなし（完全に劣化ゼロ）
  • 処理速度: 非常に高速（ファイルのコピーと同等）
  • 出力ファイルサイズ: 元のファイルとほぼ同じ
  • 出力形式: MP4（元のm4vファイルと同じ名前で.mp4拡張子）

📋 基本コマンド

ffmpeg -i input.m4v -c copy output.mp4

📋 一括変換コマンド

for file in *.m4v; do
  ffmpeg -i "$file" -c copy "${file%.m4v}.mp4"
done

📋 エイリアス定義（zshの場合）

v2p() {
  for file in *.m4v; do
    ffmpeg -i "$file" -c copy "${file%.m4v}.mp4"
  done
}

基本コマンド（QuickTime対応）

igdl "URL"

QuickTimeで再生できない

エラー：This environment is externally managed

原因: 直接pipを使用しようとしている

解決方法: pipxを使用してください（pipx install yt-dlp）

⚠️ 注意事項

  • 利用規約: 他人の著作物の無断ダウンロードは違法です。Instagramの利用規約を確認してください

🎼 twエイリアスの詳細

エイリアスコマンド

tw

🔍 コマンドの動作説明

複合機でスキャンしたトレヴァー・ワイのJPEGを1枚ずつA4サイズ（2050x2980px）に整形し、Piascoreで読みやすいPDFに変換します。中間ファイル（out_*.jpg）を生成してから最終PDFを作成します。

1️⃣ JPEGファイルをループ処理
for f in *.jpg *.jpeg(N); do ... done

カレントディレクトリの .jpg / .jpeg を1枚ずつ処理します。(N) はファイルが存在しない場合エラーを出さないzshのグロブ修飾子です。

2️⃣ リサイズ
-resize 90%

スキャン画像を90%に縮小して余白スペースを確保します。

3️⃣ コントラスト調整
-level 20%,95%

黒点を20%、白点を95%に設定。スキャン時の薄いグレー背景を飛ばして、楽譜の文字を見やすくします。

4️⃣ A4キャンバスにセンタリング
-gravity center -extent 2050x2980

2050x2980px（A4縦相当）の白いキャンバスに画像をセンタリングして配置します。全ページのサイズを統一します。

5️⃣ 中間ファイル出力
out_${f%.*}.jpg

整形済み画像を out_元ファイル名.jpg として保存します（例: out_001.jpg）。

6️⃣ PDF化
magick out_*.jpg -compress JPEG -quality 30 output.pdf

中間ファイルをまとめてPDF化。JPEG圧縮・品質30でPiascoreに最適なファイルサイズに仕上げます。

💡 使用方法

  1. 複合機でスキャンしたJPEGファイルを同じフォルダに用意
  2. ターミナルでそのフォルダに移動して tw を実行
  3. 中間ファイル（out_*.jpg）と output.pdf が生成される
  4. output.pdfをPiascoreにインポート

⚙️ 変換設定

  • リサイズ: 90%
  • レベル補正: 黒点20% / 白点95%
  • 出力サイズ: 2050×2980px（A4縦）
  • PDF圧縮: JPEG 品質30
  • 出力ファイル: output.pdf

📋 元のコード

alias tw='for f in *.jpg *.jpeg(N); do magick "$f" -resize 90% -level 20%,95% -gravity center -extent 2050x2980 "out_${f%.*}.jpg"; done && magick out_*.jpg -compress JPEG -quality 30 output.pdf'

📄 gz2pエイリアスの詳細

エイリアスコマンド

gz2p

🔍 コマンドの動作説明

カレントディレクトリの画像ファイル（HEIC/JPG/PNG）をJPEGに変換し、ファイル名順にソートして1つのPDFに結合します。一時ファイルは処理後に自動削除され、出力PDFは output_YYYYMMDD.pdf の形式で保存されます。

💡 使用方法

  1. 結合したい画像ファイルが入ったフォルダへ移動
  2. ターミナルでgz2pコマンドを実行
  3. output_YYYYMMDD.pdf が同じフォルダに出力されます

⚙️ 変換設定

  • 対応形式: HEIC、JPG、PNG（大文字・小文字どちらも対応）
  • JPEG圧縮品質: 90%（sips使用）
  • ソート: 数値順 (sort -V)
  • PDF結合: macOS内蔵の Automator アクションを使用
  • 出力ファイル名: output_YYYYMMDD.pdf（実行日付）
  • 一時ファイル: 処理後に自動削除

📋 元のエイリアス設定

alias gz2p='tmpdir=$(mktemp -d); outname="output_$(date +%Y%m%d).pdf"; setopt nullglob; files=(*.heic *.HEIC *.jpg *.JPG *.jpeg *.JPEG *.png *.PNG); unsetopt nullglob; sorted=($(printf '"'"'%s\n'"'"' "${files[@]}" | sort -V)); pdfs=(); for f in "${sorted[@]}"; do tmpjpg="$tmpdir/${f%.*}.jpg"; tmppdf="$tmpdir/${f%.*}.pdf"; sips -s format jpeg -s formatOptions 90 "$f" --out "$tmpjpg" > /dev/null 2>&1; cupsfilter -i image/jpeg -m application/pdf "$tmpjpg" > "$tmppdf" 2>/dev/null; pdfs+=("$tmppdf"); done; "/System/Library/Automator/Combine PDF Pages.action/Contents/MacOS/join" -o "$outname" "${pdfs[@]}"; rm -rf "$tmpdir"; echo "完了: $outname"'

🖼️ rs15関数の詳細

実行コマンド

rs15

🔍 コマンドの動作説明

カレントディレクトリの *.heic / *.HEIC ファイルを最長辺 1500px・解像度 72 DPI にリサイズします。macOS 標準の sips を使用するため、ImageMagick 不要。元ファイルはそのまま残し、resized_ プレフィックスを付けて出力します。

💡 使用方法

  1. リサイズしたい HEIC 画像が含まれるフォルダへ移動
  2. ターミナルでrs15を実行
  3. 各ファイルの resized_元ファイル名.heic が生成されます

⚙️ 変換設定

  • 入力形式: HEIC（*.heic / *.HEIC）
  • 最長辺: 1500px（アスペクト比を保持）
  • 解像度: 72 DPI（dpiWidth / dpiHeight）
  • 出力: resized_元ファイル名.heic（元ファイルは保持）
  • 必要ツール: sips（macOS 標準搭載、追加インストール不要）

📋 関数の登録コマンド

cat >> ~/.zshrc << 'EOF'
function rs15() {
  setopt nullglob
  for file in *.heic *.HEIC; do
    sips -Z 1500 -s dpiWidth 72 -s dpiHeight 72 "$file" --out "resized_${file}"
  done
  unsetopt nullglob
}
EOF
source ~/.zshrc
