☕ update-javaエイリアスの詳細

エイリアスコマンド

update-java

🔍 コマンドの動作説明

Apple Silicon（M1/M2）とIntel Macを自動判定し、適切なJavaをインストール・更新するコマンドです。

💡 使用方法

  1. ターミナルでupdate-javaコマンドを実行
  2. システムのアーキテクチャが自動判定される
  3. 該当するJavaをインストール・更新

⚙️ 特徴

  • Apple Silicon (ARM64)とIntel (x86_64)を自動判定
  • 各アーキテクチャに最適なJavaをインストール
  • 既にインストール済みの場合は更新を実行

📋 元のコード

update-java() {
  if [ -d "/opt/homebrew/opt/openjdk" ]; then
    echo "Apple Silicon Macとして実行"
    brew install openjdk && sudo ln -sfn /opt/homebrew/opt/openjdk/libexec/openjdk.jdk /Library/Java/JavaVirtualMachines/openjdk.jdk && echo "export PATH=\"/opt/homebrew/opt/openjdk/bin:\$PATH\"" >> ~/.zshrc && source ~/.zshrc && java -version
  else
    echo "Intel Macとして実行"
    brew install openjdk && sudo ln -sfn /usr/local/opt/openjdk/libexec/openjdk.jdk /Library/Java/JavaVirtualMachines/openjdk.jdk && echo "export PATH=\"/usr/local/opt/openjdk/bin:\$PATH\"" >> ~/.zshrc && source ~/.zshrc && java -version
  fi
}

🖼️ herowebpエイリアスの詳細

エイリアスコマンド

herowebp

🔍 コマンドの動作説明

jpg/jpeg/png/heic形式の画像を3つのサイズのWebP形式に自動変換します。ヒーロー画像などレスポンシブで複数サイズが必要な場合に便利です。

💡 使用方法

  1. カレントディレクトリの画像を配置
  2. ターミナルでherowebpコマンドを実行
  3. 以下の3サイズのWebP画像が自動生成されます：
    • 1920x1080px
    • 1200x675px
    • 800x450px

⚙️ 対応形式

  • 入力形式: jpg, jpeg, png, heic
  • 出力形式: WebP (複数サイズ)

📋 元のコード

herowebp() {
  local quality=50
  local method=6
  if [ ! -d "$HOME/editing" ]; then
    echo "Error: ~/editing directory not found"
    return 1
  fi
  cd "$HOME/editing" || return 1
  for img in *.{jpg,jpeg,png,heic}; do
    if [ -f "$img" ]; then
      base="${img%.*}"
      echo "Processing: $img"
      magick "$img" -resize 1920x1080 -quality $quality -define webp:method=$method "${base}_1920x1080.webp"
      magick "$img" -resize 1200x675 -quality $quality -define webp:method=$method "${base}_1200x675.webp"
      magick "$img" -resize 800x450 -quality $quality -define webp:method=$method "${base}_800x450.webp"
      echo "✓ Generated WebP files for: $img"
    fi
  done
}

🖼️ webp50エイリアスの詳細

エイリアスコマンド

webp50

🔍 コマンドの動作説明

jpg/jpeg/png/heic形式を自動検出して、高圧縮率のWebPに変換します。品質50とmethod=6（最高品質の圧縮）で最適なバランスを実現します。

💡 使用方法

  1. カレントディレクトリの画像を配置
  2. ターミナルでwebp50コマンドを実行
  3. 同じフォルダにWebP形式の画像が生成されます

⚙️ 圧縮オプション

  • quality=50: 品質レベル (0-100、50は標準的な圧縮)
  • method=6: 圧縮方法 (0-6、6は最高品質)

📋 元のコード

webp50() {
  local quality=50
  local method=6
  local count=0
  if [ ! -d "$HOME/editing" ]; then
    echo "Error: ~/editing directory not found"
    return 1
  fi
  cd "$HOME/editing" || return 1
  for file in *.{jpg,jpeg,png,heic}; do
    if [ -f "$file" ]; then
      local base="${file%.*}"
      if [ -f "${base}.webp" ]; then
        echo "⊘ Already exists: ${base}.webp (skipped)"
        continue
      fi
      echo "Converting: $file → ${base}.webp"
      magick "$file" -quality $quality -define webp:method=$method "${base}.webp"
      if [ $? -eq 0 ]; then
        echo "✓ Converted: $file"
        ((count++))
      else
        echo "✗ Failed to convert: $file"
      fi
    fi
  done
  echo ""
  if [ $count -gt 0 ]; then
    echo "✓ Successfully converted $count file(s) to WebP"
  else
    echo "⊘ No new files to convert"
  fi
}

🖼️ img2jpegエイリアスの詳細

エイリアスコマンド

img2jpeg

🔍 コマンドの動作説明

複数の画像形式をJPEGに統一して変換します。長辺を1080pxにリサイズするため、ウェブ用途に最適なサイズになります。

💡 使用方法

  1. カレントディレクトリの画像を配置
  2. ターミナルでimg2jpegコマンドを実行
  3. 同じフォルダにJPEG形式（長辺1080px）の画像が生成されます

⚙️ 変換設定

  • 出力形式: JPEG
  • 最大長辺: 1080px
  • アスペクト比: 維持

📋 元のコード

img2jpeg() {
  setopt nullglob
  local count=0
  local failed=0
  local total=0

  for file in *.tiff *.tif *.png *.jpeg *.jpg *.heic; do
    [ -f "$file" ] || continue
    local base="${file%.*}"
    ((total++))

    echo "Converting: $file → ${base}.jpg"
    sips -s format jpeg -Z 1080 "$file" --out "${base}.jpg" &>/dev/null

    if [ $? -eq 0 ]; then
      echo "✓ $(basename "$file")"
      ((count++))
    else
      echo "✗ $(basename "$file")"
      ((failed++))
    fi
  done

  echo ""
  if [ $total -eq 0 ]; then
    echo "⊘ No image files found (tiff, tif, png, jpeg, jpg, heic)"
  elif [ $count -gt 0 ]; then
    echo "✓ Successfully converted $count file(s) to JPEG"
  fi
  if [ $failed -gt 0 ]; then
    echo "✗ Failed to convert $failed file(s)"
  fi
}

🖼️ geniconエイリアスの詳細

エイリアスコマンド

genicon

🔍 コマンドの動作説明

editing フォルダ内のPNG画像から、3種類のサイズのアイコンを一度に生成します。ウェブサイトやアプリケーションで必要なすべてのアイコンサイズを自動作成できます。

💡 使用方法

  1. カレントディレクトリのPNG画像を配置
  2. ターミナルでgeniconコマンドを実行
  3. 以下の3種類のWebPアイコンが同じフォルダに生成されます

⚙️ 生成されるサイズ

  • 256x256px (大サイズ / ファビコン用)
  • 180x180px (中サイズ / スマートフォン用)
  • 32x32px (小サイズ / 補助的なアイコン用)

※ すべてWebP形式で出力されます

📋 元のコード

generateicons() {
  if [ ! -d "$HOME/editing" ]; then
    echo "✗ エラー: ~/editing ディレクトリが見つかりません"
    return 1
  fi
  cd "$HOME/editing" || return 1
  local count=0
  for file in *.png; do
    if [ -f "$file" ]; then
      local base="${file%.*}"
      echo "処理中: $file"
      magick "$file" -resize 256x256 "${base}_256x256.png"
      magick "$file" -resize 180x180 "${base}_180x180.png"
      magick "$file" -resize 32x32 "${base}_32x32.png"
      echo "✓ ${base}_256x256.png"
      echo "  ✓ ${base}_180x180.png"
      echo "  ✓ ${base}_32x32.png"
      ((count++))
    fi
  done
  echo ""
  if [ $count -gt 0 ]; then
    echo "✓ 完了: $count ファイルの変換が完了しました"
  else
    echo "⊘ PNG ファイルが見つかりません"
  fi
}

🖨️ h2pエイリアスの詳細

エイリアスコマンド

h2p

🔍 機能説明

HEIC形式の画像をJPG形式に変換してから、SHARPカラープリンタで自動的にカラー印刷します。画像の向き（縦横）を自動判定し、用紙に最適な方向で印刷します。

💡 使用方法

基本的な使用手順：

  1. 印刷したい HEIC ファイルがあるフォルダでターミナルを開く（またはカレントディレクトリに HEIC ファイルを配置）
  2. 必要に応じて hrsz() で画像をトリム・リサイズ（2480x3508 または 3508x2480）
  3. ターミナルで h2p を実行

🔄 処理フロー

  • HEIC → JPG 変換：ImageMagick (magick) を使用して自動回転対応で変換
  • 画像サイズ判定：変換後のJPGサイズから向きを自動判定
    • 横長（幅 > 高さ）→ 横向き印刷
    • 縦長（幅 ≤ 高さ）→ 縦向き印刷
  • 用紙いっぱい印刷：fit-to-page オプションで拡大印刷
  • カラー印刷：ColorModel=RGB を指定
  • 自動クリーンアップ：印刷完了後、一時JPGファイルを削除

📋 エイリアス定義

h2p() {
  setopt nullglob
  for file in ./*.heic; do
    [ -f "$file" ] || continue
    jpg="${file%.heic}.jpg"
    if ! magick "$file" -auto-orient "$jpg"; then
      echo "✗ エラー: $(basename "$file") の変換に失敗"
      rm -f "$jpg"
      continue
    fi
    dimensions=$(magick identify -format "%w %h" "$jpg")
    width=$(echo $dimensions | cut -d' ' -f1)
    height=$(echo $dimensions | cut -d' ' -f2)
    if [ "$width" -gt "$height" ]; then
      orientation=4
    else
      orientation=3
    fi
    if lp -d SHARP_BP_60C26__3504320300_ -o ColorModel=RGB -o fit-to-page -o orientation-requested=$orientation "$jpg" &>/dev/null; then
      sleep 3
      rm -f "$jpg"
      echo "✓ 処理完了: $(basename "$file")"
    else
      echo "✗ エラー: $(basename "$file") の印刷に失敗"
      rm -f "$jpg"
    fi
  done
}

🖼️ hrszエイリアスの詳細

エイリアスコマンド

hrsz

🔍 機能説明

HEIC形式の画像を正規化サイズにリサイズ・トリムします。画像の向き（縦横）を自動判定し、アスペクト比を損なわないように中央からトリムして、3508x2480 または 2480x3508 のサイズに統一します。

💡 使用方法

基本的な使用手順：

  1. HEIC ファイルがあるフォルダで hrsz を実行
  2. 画像の向きに応じて自動的にサイズを決定
  3. output/ フォルダに処理済み画像を出力

🔄 処理フロー

  • サイズ判定：元画像の幅と高さを取得
    • 横長（幅 > 高さ）→ 3508x2480 にリサイズ・トリム
    • 縦長（幅 ≤ 高さ）→ 2480x3508 にリサイズ・トリム
  • 高品質リサイズ：-density 300 で 300 DPI に設定
  • 中央トリム：-gravity Center で中央からトリム（アスペクト比を損なわない）
  • 拡大対応：-resize ^ で小さい画像も拡大可能
  • ファイル保持：元のファイル名で output/ に出力

📋 エイリアス定義

hrsz() {
  output_dir="output"
  mkdir -p "$output_dir"
  for file in *.heic; do
    width=$(magick identify -format "%w" "$file")
    height=$(magick identify -format "%h" "$file")
    if [ "$width" -gt "$height" ]; then
      magick "$file" -density 300 -gravity Center -resize 3508x2480^ -crop 3508x2480+0+0 +repage "$output_dir/$file"
    else
      magick "$file" -density 300 -gravity Center -resize 2480x3508^ -crop 2480x3508+0+0 +repage "$output_dir/$file"
    fi
    echo "✓ 処理完了: $file"
  done
  echo "✅ 全ファイルが $output_dir フォルダに出力されました"
}

📚 関連コマンド

h2p - 印刷用エイリアス。output/ フォルダの処理済み画像を JPEG に変換して SHARP カラープリンタで印刷します。

💻 使用例

例1: 複数の HEIC ファイルを処理

$ cd <画像フォルダ>
$ hrsz
✓ 処理完了: photo1.heic
✓ 処理完了: photo2.heic
✓ 処理完了: photo3.heic
✅ 全ファイルが output フォルダに出力されました

例2: hrsz で前処理して、h2p で印刷

$ cd <画像フォルダ>
$ hrsz
✅ 全ファイルが output フォルダに出力されました
$ cp output/*.heic ~/editing/
$ h2p
✓ 処理完了: photo1.heic
✓ 処理完了: photo2.heic

⚠️ 注意点

  • 元の HEIC ファイルはそのまま保持されます（output/ フォルダに複製）
  • 小さい画像も拡大されるため、元の解像度が低い場合は品質が低下する可能性があります
  • output/ フォルダがない場合は自動的に作成されます
  • 出力形式は元のファイル拡張子を保持します（HEIC → HEIC）

🔧 技術詳細

  • -density 300：300 DPI で高品質印刷対応
  • -gravity Center：中央を基準にトリム
  • -resize ^：小さい画像も拡大対応
  • -crop：余白を均等に削除
  • +repage：トリム後の画像メタデータを更新

🖼️ hrsz500エイリアスの詳細

エイリアスコマンド

hrsz500

🔍 機能説明

JPG/PNG/HEIC ファイルを 500x500 ピクセル、72 PPI に統一して iTunes アートワーク用に変換・リサイズします。下位フォルダ内のファイルも含めて全て処理します。背景色は白で統一され、不足部分は白で埋められます。元のJPEG/PNGファイルはデスクトップの original_images フォルダに移動され、処理完了ファイルは output フォルダに出力されます。

💡 使用方法

基本的な使用手順：

  1. 画像ファイルがあるフォルダで hrsz500 を実行
  2. フォルダ内および下位フォルダの全ての画像が処理される
  3. 処理されたファイルは output フォルダに 500x500、72 PPI のHEIC形式で出力
  4. 元のJPEG/PNGファイルは ~/Desktop/original_images/ に移動

🔄 処理フロー

  • ディレクトリ作成：~/Desktop/original_images と output フォルダを作成
  • JPEG/PNG処理：
    • 500x500 ピクセルにリサイズ・クロップ（中心基準、背景色は白）
    • 72 PPI で HEIC形式に変換（品質75）
    • output フォルダに出力
    • 元のファイルを original_images/ に移動
  • 既存HEIC処理：
    • 500x500 ピクセルにリサイズ・クロップ（背景色は白）
    • 72 PPI で品質75に統一
    • output フォルダに出力
  • 下位フォルダ対応：find コマンドで再帰的に全ファイルを処理

📋 エイリアス定義

hrsz500() {
  mkdir -p ~/Desktop/original_images
  mkdir -p output

  find . -type f \( -iname "*.jpeg" -o -iname "*.jpg" -o -iname "*.png" \) | while read file; do
    filename=$(basename "$file" | sed 's/\.[^.]*$//')
    output_file="output/${filename}.heic"
    magick "$file" -resize 500x500 -gravity center -background white -extent 500x500 -density 72 -quality 75 "$output_file"
    mv "$file" ~/Desktop/original_images/
    echo "Converted & resized: $(basename "$file") → $(basename "$output_file") (original moved)"
  done

  find . -type f -iname "*.heic" | while read file; do
    filename=$(basename "$file")
    temp_file="output/${filename}.tmp"
    magick "$file" -resize 500x500 -gravity center -background white -extent 500x500 -density 72 -quality 75 "$temp_file"
    mv "$temp_file" "output/${filename}"
    echo "Resized: output/$(basename "$file")"
  done
}

💻 使用例

例1: JPEG/PNG/HEICが混在したフォルダを処理

$ cd ~/editing
$ hrsz500
Converted & resized: photo1.jpg → output/photo1.heic (original moved)
Converted & resized: photo2.png → output/photo2.heic (original moved)
Resized: output/photo3.heic

⚠️ 注意点

  • 全ての処理済みファイルは output フォルダに出力されます
  • 元のJPEG/PNGファイルは ~/Desktop/original_images/ に移動されます
  • 既存のHEICファイルも500x500、72 PPIにリサイズされます
  • 長辺が500px以下の小さい画像は拡大されます
  • 正方形でない画像は中央からクロップされ、不足部分は白で埋められます
  • 下位フォルダ内のファイルも再帰的に処理されます
  • quality 75 で圧縮されるため、完全な無損失ではありません

🔧 技術詳細

  • -resize 500x500：アスペクト比を保持してリサイズ
  • -gravity center：中央を基準にクロップ・配置
  • -background white：背景色を白に設定
  • -extent 500x500：500x500のキャンバスに配置
  • -density 72：72 PPI (ピクセルパーインチ) に設定
  • -quality 75：品質75で圧縮
  • find . -type f：カレントディレクトリと下位フォルダの全ファイルを再帰的に処理

  • ImageMagick：brew install imagemagick

🔢 rnエイリアスの詳細

エイリアスコマンド

rn

🔍 コマンドの動作説明

カレントディレクトリにある画像ファイル（jpg/jpeg/png/tiff/tif/gif/webp/bmp）を対話形式で連番リネームします。開始番号・前後に付ける文字列を指定でき、実行前にプレビューで確認できます。

💡 使用方法

  1. リネームしたい画像があるフォルダに移動
  2. ターミナルでrnを実行
  3. 開始番号を入力して確認
  4. 番号の前後に文字列を追加するか選択（任意）
  5. プレビューを確認してリネーム実行

⚙️ 主な機能

  • 対応形式：jpg / jpeg / png / tiff / tif / gif / webp / bmp
  • 開始番号の入力バリデーション（数字以外はエラー）
  • 番号の前（prefix）または後（suffix）に任意文字列を追加可能
  • リネーム前にプレビュー表示・確認プロンプト
  • mv -iで既存ファイルへの上書きを防止

📋 元のコード

rn() {
  # 開始番号を訊く
  while true; do
    echo -n "開始番号を入力してください: "
    read start
    if [[ -z "$start" || ! "$start" =~ ^[0-9]+$ ]]; then
      echo "エラー: 数字を入力してください。"
      continue
    fi
    echo -n "開始番号は「${start}」でよいですか？ (y/n): "
    read confirm_start
    [[ "$confirm_start" == "y" ]] && break
  done

  # 前後に文字列を付けるか訊く
  local prefix=""
  local suffix=""
  echo -n "番号の前後に文字列を追加しますか？ (y/n): "
  read add_str
  if [[ "$add_str" == "y" ]]; then
    echo -n "文字列を入力してください: "
    read str
    echo -n "番号の「前」ですか「後」ですか？ (前/後): "
    read pos
    if [[ "$pos" == "前" ]]; then
      prefix="$str"
    else
      suffix="$str"
    fi
  fi

  # 対象ファイルを画像から自動検出
  local img_files=()
  for f in *(N-.); do
    [[ "$f" =~ \.(jpg|jpeg|png|tiff|tif|gif|webp|bmp)$ ]] && img_files+=("$f")
  done

  if [[ ${#img_files[@]} -eq 0 ]]; then
    echo "対象の画像ファイルが見つかりません。" >&2
    return 1
  fi

  # プレビュー表示
  echo ""
  echo "【プレビュー】"
  local n=$start
  for f in "${img_files[@]}"; do
    local ext="${f##*.}"
    local new_name="${prefix}${n}${suffix}.${ext}"
    echo "$f → $new_name"
    (( n++ ))
  done

  # 確認
  echo ""
  echo -n "このままリネームしますか？ (y/n): "
  read confirm
  if [[ "$confirm" != "y" ]]; then
    echo "キャンセルしました。"
    return 0
  fi

  # 実行
  n=$start
  for f in "${img_files[@]}"; do
    local ext="${f##*.}"
    local new_name="${prefix}${n}${suffix}.${ext}"
    mv -i -- "$f" "$new_name"
    (( n++ ))
  done
  echo "リネーム完了しました。"
}

📋 実装コード

printpdfbw() {
  local count=0
  local found=0

  for pdf in *.pdf; do
    if [ -f "$pdf" ]; then
      found=1
      lp -d SHARP_BP_60C26__3504320300_ -o ColorModel=Gray -o scaling=100 "$pdf"
      echo "✓ 印刷中: $pdf"
      ((count++))
    fi
  done

  if [ $found -eq 0 ]; then
    echo "✗ エラー: カレントディレクトリに PDF ファイルが見つかりません"
    return 1
  fi

  echo ""
  echo "✅ 完了: $count 個の PDF ファイルを白黒で印刷キューに追加しました"
}

📦 zpエイリアスの詳細

エイリアスコマンド

zp

🔍 コマンドの動作説明

カレントディレクトリをCP932（Windows互換）エンコーディングでZIP化します。Windowsで日本語ファイル名が文字化けしないZIPファイルを作成するために最適です。

💡 使用方法

  1. 圧縮したいフォルダで右クリック
  2. 「フォルダに新規ターミナル」を選択
  3. ターミナルでzpと入力
  4. カレントディレクトリが1つのZIPファイルに圧縮されます

⚙️ 特徴

  • エンコーディング: CP932（Windows互換）
  • 日本語ファイル名: 文字化けしない
  • 日本語フォルダ名: 対応
  • 特殊文字対応: ㈱、①②③など

📋 元のコード

zipthis_cp932() {
  local FOLDER_NAME=$(basename "$PWD")
  local OUTPUT_NAME="${1:-$FOLDER_NAME.zip}"
  local output_dir=$(dirname "$PWD")
  python3 << PYTHON_EOF
import zipfile
import os
import sys

def sanitize_filename(filename):
    filename = filename.replace('・', '_')
    filename = filename.replace('　', '')
    return filename

try:
    folder_path = "$PWD"
    zip_path = "$output_dir/$OUTPUT_NAME"
    with zipfile.ZipFile(zip_path, 'w', zipfile.ZIP_DEFLATED) as zf:
        for root, dirs, files in os.walk(folder_path):
            for file in files:
                file_path = os.path.join(root, file)
                arcname = os.path.join(os.path.basename(folder_path), os.path.relpath(file_path, folder_path))
                arcname = sanitize_filename(arcname)
                zinfo = zipfile.ZipInfo(filename=arcname.encode('cp932').decode('cp932'))
                zinfo.compress_type = zipfile.ZIP_DEFLATED
                with open(file_path, 'rb') as f:
                    zf.writestr(zinfo, f.read())
    print(f"✓ Created: ../{os.path.basename(zip_path)}")
    sys.exit(0)
except Exception as e:
    print(f"✗ Error: {e}")
    sys.exit(1)
PYTHON_EOF
}

📦 zpcp932エイリアスの詳細

エイリアスコマンド

zpcp932

🔍 コマンドの動作説明

カレントディレクトリ自体を1つのZIPファイルにまとめます。ZIPファイルは親ディレクトリに作成されます。

💡 使用方法

  1. ZIP化したいディレクトリに移動（cd コマンドで対象ディレクトリへ）
  2. ターミナルでzpcp932コマンドを実行
  3. カレントディレクトリと同名のZIPファイルが親ディレクトリに作成されます

⚙️ 処理内容

  • 処理対象: カレントディレクトリ全体
  • エンコーディング: CP932（Windows互換）
  • 出力先: 親ディレクトリ（カレントディレクトリ名.zip）
  • 既にZIPが存在する場合はスキップ

📋 元のコード

zpcp932() {
  local dirname=$(basename "$PWD")
  local zipname="${dirname}.zip"
  local zip_path="$(dirname "$PWD")/$zipname"

  # 既にZIPが存在する場合はスキップ
  if [ -f "$zip_path" ]; then
    echo "⊘ スキップ: $dirname (既にZIPが存在します: $zip_path)"
    return 1
  fi

  echo "処理中: $dirname → $zip_path"

  # ZIPフォーマットをバイナリで直接書き出す（UTF-8フラグ強制セットを回避）
  python3 << PYTHON_EOF
import os, sys, struct, zlib, time

try:
    folder_path = "$PWD"
    zip_path = "$zip_path"
    parent_path = os.path.dirname(folder_path)

    entries = []

    with open(zip_path, 'wb') as zf:
        for root, dirs, files in os.walk(folder_path):
            dirs.sort()
            for file in sorted(files):
                file_path = os.path.join(root, file)
                arcname = os.path.relpath(file_path, parent_path)
                arcname_bytes = arcname.encode('cp932')

                with open(file_path, 'rb') as f:
                    data = f.read()

                compressed = zlib.compress(data, 6)[2:-4]  # deflate
                crc = zlib.crc32(data) & 0xFFFFFFFF
                t = time.localtime()
                dostime = (t.tm_hour << 11) | (t.tm_min << 5) | (t.tm_sec // 2)
                dosdate = ((t.tm_year - 1980) << 9) | (t.tm_mon << 5) | t.tm_mday

                offset = zf.tell()

                # Local file header
                lh = struct.pack('<4sHHHHHIIIHH',
                    b'PK\x03\x04',
                    20,          # version needed
                    0,           # flags: 0 (no UTF-8 flag)
                    8,           # compression: deflate
                    dostime, dosdate,
                    crc,
                    len(compressed),
                    len(data),
                    len(arcname_bytes),
                    0            # extra field length
                )
                zf.write(lh)
                zf.write(arcname_bytes)
                zf.write(compressed)

                entries.append((arcname_bytes, crc, len(compressed), len(data), dostime, dosdate, offset))

        # Central directory
        cd_offset = zf.tell()
        for arcname_bytes, crc, comp_size, uncomp_size, dostime, dosdate, offset in entries:
            cd = struct.pack('<4sHHHHHHIIIHHHHHII',
                b'PK\x01\x02',
                20,          # version made by
                20,          # version needed
                0,           # flags: 0
                8,           # compression
                dostime, dosdate,
                crc,
                comp_size,
                uncomp_size,
                len(arcname_bytes),
                0, 0, 0, 0,  # extra, comment, disk, attr
                0,           # external attr
                offset
            )
            zf.write(cd)
            zf.write(arcname_bytes)

        # End of central directory
        cd_size = zf.tell() - cd_offset
        eocd = struct.pack('<4sHHHHIIH',
            b'PK\x05\x06',
            0, 0,
            len(entries), len(entries),
            cd_size, cd_offset,
            0
        )
        zf.write(eocd)

    sys.exit(0)
except Exception as e:
    print(f"✗ エラー: {e}")
    sys.exit(1)
PYTHON_EOF

  if [ $? -eq 0 ]; then
    echo "✓ 完了: $zip_path"
  else
    echo "✗ エラー: $dirname の変換に失敗しました"
  fi
}

📄 pdf2pngエイリアスの詳細

エイリアスコマンド

pdf2png

🔍 コマンドの動作説明

カレントディレクトリのPDFファイルを300dpi解像度でPNG画像に変換します。ロスレス形式で高品質を維持しながら、最大幅1200pxにリサイズして変換できます。

💡 使用方法

  1. カレントディレクトリのPDFを配置
  2. ターミナルでpdf2pngコマンドを実行
  3. 各PDFページがPNG画像になって同じフォルダに出力されます

⚙️ 変換設定

  • 解像度: 300dpi (高解像度)
  • 品質: 85%
  • リサイズ: 1200x (最大幅1200px)
  • 出力形式: PNG (ロスレス)

📋 元のコード

pdf2png() {
  for file in "$HOME/editing"/*.pdf; do
    [ -f "$file" ] || continue
    output="${file%.pdf}.png"
    magick -density 300 "$file" -flatten -background white -resize 1200x "$output" && \
    echo "✓ $(basename "$file") → $(basename "$output")" || \
    echo "✗ $(basename "$file")"
  done
}

🎬 mvエイリアスの詳細

エイリアスコマンド

cd ~/editing && ls -1 *.mp4 *.m4v 2>/dev/null | awk '{print "file " $0}' > filelist.txt && ffmpeg -f concat -safe 0 -i filelist.txt -c:v copy -c:a copy output.mp4 && ffmpeg -i output.mp4 -c:v copy -c:a copy -movflags +faststart -color_range tv output_fixed.mp4 && mv output_fixed.mp4 output.mp4 && rm filelist.txt

🔍 コマンドの動作説明

1️⃣ 作業フォルダへ移動
cd ~/editing

カレントディレクトリで処理します。

2️⃣ ファイル検索
ls -1 *.mp4 *.m4v 2>/dev/null

カレントディレクトリのすべてのmp4とm4vファイルを取得します。-1フラグで1ファイルずつ1行に表示。2>/dev/nullはエラーメッセージを非表示にします。

3️⃣ ファイルリスト生成
awk '{print "file " $0}' > filelist.txt

各ファイルを1行ずつ処理し、FFmpegのconcat形式（file ファイル名）でfilelist.txtに出力します。

4️⃣ MP4/M4Vファイル結合
ffmpeg -f concat -safe 0 -i filelist.txt -c:v copy -c:a copy output.mp4

FFmpegを使用してファイルリストを読み込み、すべてのmp4/m4vファイルを1つのoutput.mp4に結合します。-c:v copy -c:a copyはビデオとオーディオを再エンコードしません。

5️⃣ QuickTime対応メタデータ修正
ffmpeg -i output.mp4 -c:v copy -c:a copy -movflags +faststart -color_range tv output_fixed.mp4

QuickTimeが正しく色を表示するようにメタデータを再適用します。-movflags +faststartでストリーミング対応、-color_range tvでテレビ標準色範囲を設定します。

6️⃣ ファイル置き換え・クリーンアップ
mv output_fixed.mp4 output.mp4 && rm filelist.txt

修正されたファイルをoutput.mp4に置き換え、一時ファイルfilelist.txtを削除します。

💡 使用方法

  1. カレントディレクトリのmp4ファイルを配置
  2. ターミナルでmp4combineコマンドを実行
  3. 同じフォルダにoutput.mp4が生成されます

⚙️ オプション説明

  • -f concat: FFmpegのconcat demuxerを使用（複数ファイルの結合用）
  • -safe 0: ファイルパスの安全性チェックを無視（日本語ファイル名対応）
  • -i filelist.txt: 結合するファイルのリストを入力
  • -c:v copy -c:a copy: ビデオ・オーディオを再エンコードせず品質保持
  • -movflags +faststart: QuickTime/ブラウザで即座に再生可能にする
  • -color_range tv: 色範囲をテレビ標準（制限範囲）に設定
  • 2>/dev/null: ファイルが見つからないエラーメッセージを非表示にする

🎬 v2pエイリアスの詳細

エイリアスコマンド

v2p

🔍 コマンドの動作説明

m4vファイルをmp4ファイルに劣化なしで変換します。m4vとmp4は同じMPEG-4コンテナフォーマットのため、再エンコードを行わずコンテナだけを変更することで、品質を損なわずに高速で変換できます。

💡 M4VとMP4について

  • M4V形式: Apple製品用のMPEG-4ビデオコンテナ（DRM保護機能付きの場合もあります）
  • MP4形式: より汎用的なMPEG-4ビデオコンテナ
  • 内部構造: 両者は本質的に同じコンテナフォーマット
  • 変換方法: -c copyオプションでコンテナだけを変更し、ビデオ・オーディオの再エンコードを行わない

💡 使用方法

フォルダを右クリックして「新規ターミナル」を選択すると、そのフォルダに移動した状態でターミナルが起動します。その後、ターミナルでv2pと入力して実行してください。

  1. m4vファイルがあるフォルダで右クリック
  2. 「新規ターミナル」を選択（フォルダに移動した状態で起動）
  3. ターミナルでv2pと入力して実行
  4. 同じフォルダにmp4ファイルが生成されます

⚙️ 処理内容

  • 対象: カレントディレクトリ内のすべてのm4vファイル
  • 処理方式: 再エンコードなし（完全に劣化ゼロ）
  • 処理速度: 非常に高速（ファイルのコピーと同等）
  • 出力ファイルサイズ: 元のファイルとほぼ同じ
  • 出力形式: MP4（元のm4vファイルと同じ名前で.mp4拡張子）

📋 基本コマンド

ffmpeg -i input.m4v -c copy output.mp4

📋 一括変換コマンド

for file in *.m4v; do
  ffmpeg -i "$file" -c copy "${file%.m4v}.mp4"
done

📋 エイリアス定義（zshの場合）

v2p() {
  for file in *.m4v; do
    ffmpeg -i "$file" -c copy "${file%.m4v}.mp4"
  done
}

📝 エイリアス登録方法

zshを使用している場合は、~/.zshrcファイルに上記のコードを追加してください。

  1. ターミナルでnano ~/.zshrcを実行
  2. ファイルの最後に上記のコードを追加
  3. Ctrl+O で保存、Enter でファイル名確認、Ctrl+X で終了
  4. ターミナルを再起動またはsource ~/.zshrcを実行

✨ このコマンドの利点

  • 完全に劣化なし: ビデオ・オーディオデータを一切変更していないため、品質は100%保持されます
  • 処理速度が非常に高速: 再エンコードを行わないため、大容量ファイルでも瞬時に変換できます
  • ファイルサイズはほぼ同じ: コンテナ情報のみの変更のため、出力ファイルサイズは元のファイルとほぼ同じです
  • Macの便利機能と組み合わせ可能: 「フォルダに新規ターミナル」機能を使えば、任意のフォルダでワンコマンドで実行できます

⚙️ オプション説明

  • -c copy: ビデオストリーム・オーディオストリームを再エンコードしない（コピーのみ）
  • *.m4v: ワイルドカードで複数のm4vファイルに対応
  • ${file%.m4v}: m4v拡張子を削除して、mp4に置き換える

📱 Instagram動画ダウンロードガイド

🎯 概要

MacのターミナルでInstagramの動画をダウンロードする方法。yt-dlpコマンドを使用して、最高品質の動画ファイルを保存できます。

📋 セットアップ（初回のみ）

1. pipx のインストール

brew install pipx

2. yt-dlp のインストール

pipx install yt-dlp

3. インストール確認

yt-dlp --version

🎬 基本的な使用方法

基本コマンド

yt-dlp "URL"

例

yt-dlp "https://www.instagram.com/p/CzBVWX8pc-C/"

⚙️ よく使うオプション

目的 コマンド
ダウンロード場所を指定 -o "~/Downloads/%(title)s.%(ext)s" "URL"
最高品質でダウンロード -f best "URL"
複数の動画をダウンロード "URL1" "URL2" "URL3"

💾 よく使うコマンド例

Downloadsフォルダに保存

yt-dlp -o "~/Downloads/%(title)s.%(ext)s" "https://www.instagram.com/p/XXXXXXXXXXXX/"

最高品質でダウンロード

yt-dlp -f best "https://www.instagram.com/p/XXXXXXXXXXXX/"

🔧 トラブルシューティング

エラー：yt-dlp: command not found

原因: pipxがインストールされていない、またはパスが通っていない

解決方法: 上記のセットアップ手順を確認して、ターミナルを再起動してください。

エラー：This environment is externally managed

原因: 直接pipを使用しようとしている

解決方法: pipxを使用してください（pipx install yt-dlp）

⚠️ 注意事項

  • 著作権: 自分がアップロードした動画のみダウンロードしてください
  • 利用規約: 他人の著作物の無断ダウンロードは違法です。Instagramの利用規約を確認してください
  • プライベートアカウント: 非公開アカウントの動画にはアクセスできません
  • 仕様変更: Instagramの仕様変更により、一時的に動作しない可能性があります
